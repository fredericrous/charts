1. The ArgoCD envsubst plugin has been installed!

2. To use the plugin in your ArgoCD Applications, specify it in the source:

   ```yaml
   apiVersion: argoproj.io/v1alpha1
   kind: Application
   spec:
     source:
       plugin:
         name: envsubst
   ```

3. Environment variables can be configured via:
   {{- if .Values.env }}
   - Direct environment variables (configured)
   {{- end }}
   {{- if .Values.envFrom }}
   - Environment from ConfigMaps/Secrets (configured)
   {{- end }}
   {{- if not .Values.env }}{{- if not .Values.envFrom }}
   - No environment configuration found. Add env or envFrom to your values.
   {{- end }}{{- end }}

4. For .env file support, create .env files in your repository:
   - Global: /.env
   - Environment-specific: /environments/${ENVIRONMENT}.env  
   - App-specific: ./.env in the app directory

5. Check the plugin logs:
   ```
   kubectl logs -n {{ .Release.Namespace }} -l app.kubernetes.io/name={{ include "argocd-envsubst-plugin.name" . }}
   ```

6. Access metrics at http://localhost:9090/metrics after port-forwarding:
   ```
   kubectl port-forward -n {{ .Release.Namespace }} svc/{{ include "argocd-envsubst-plugin.fullname" . }} 9090:9090
   ```

For more information, visit: https://github.com/fredericrous/argocd-plugin-envsubst
